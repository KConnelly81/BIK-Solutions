BIK Business Toolkit

Version 0.10 Beta 16 July 2026 — AI Project Health Dashboard

Sprint 4 replaces the static project list with an AI Project Health Dashboard — a proactive, rules-based monitoring system that watches every project and surfaces the next best action without the builder having to search for problems.

New in this release
  • ● Project Health Dashboard — each project gets a health card showing its status, contract value, active alerts, and a suggested next action. Cards are colour-coded: green (healthy), amber (needs attention), red (critical).
  • ● Health Engine — deterministic rules engine (health-engine.js) evaluates every project against its document records. Architecture is future-ready for AI recommendations — the rules engine output shape is the stable contract; swapping to a model endpoint requires no UI changes.
  • ● Running Contract Value — automatically calculates original contract (approved quote) + approved variations = current contract total, with GST estimate. Updates in real time as documents are approved.
  • ● Smart alerts with one-click actions — six alert categories: pending approval (overdue), approval awaiting, missing site diary, quote expiry, no quote on file, quote not yet sent. Every alert has a direct action button: Send Reminder, Create Site Diary, Review Quote, Create Quote, Open Document.
  • ● Pre-written follow-up emails — approval chasers (standard and urgent) are pre-written and pre-addressed. Builder clicks "Send follow-up" and the email client opens with reference, days outstanding, and professional language already composed.
  • ● Summary bar — active project count, total contract value across all projects, and total items needing attention at a glance.
  • ● Project detail page — dedicated project.html with full project timeline, documents by type, contract summary, smart actions, and coming-soon stubs for Client Portal, Digital Signatures, Progress Claims, Invoices, Site Photos, Voice Notes, and Attachments.
  • ● Hero personalisation — dashboard greeting updates based on project health state: "2 projects need your attention today" vs "All 4 projects looking healthy."
Architecture notes
  • ● js/toolkit/health-engine.js — pure module, no DOM, no side effects. HEALTH_ENGINE_SWAP_POINT comment marks where evaluateProject() would be replaced with an AI API call.
  • ● js/toolkit/project-store.js — PROJECT_STORAGE_POINT marks the swap point for Supabase backend in v2.
  • ● js/toolkit/document-history.js — DOCUMENT_HISTORY_STORAGE_POINT marks the swap point for server-side document storage.
Version 0.9 Beta 15 July 2026 — First public beta

The first beta release of BIK Business Toolkit. Five professional document tools, AI writing assistance, and a complete document management workflow — built specifically for Australian builders and tradies. This release is ready for real-world use on Variation Notices. The remaining four tools are available and functional, with further polish planned before full launch.

New in this release
  • ● Variation Notice Generator — complete, production-quality tool. Auto-numbered VN references, cost and time extension calculations with GST, AI writing modes, PDF export, email workflow, draft autosave, and document history.
  • ● Quote Builder — dynamic line items pricing table, GST calculations, inclusions/exclusions, validity period, payment schedule, and AI writing assistance.
  • ● Scope of Works — comprehensive scope documentation with site details, materials, inclusions, exclusions, authority requirements, and contract conditions.
  • ● Site Diary — daily site records including weather, workforce, works completed, deliveries, incidents, inspections, and sub-contractor notes.
  • ● Defect Report — structured defect identification with severity levels, location, access requirements, trade responsible, and rectification instructions.
  • ● AI Writer — six writing modes: Rewrite Professionally, Strengthen for Contract, Spell & Grammar, Simplify for Client, Formal Version, Plain English. Uses your own Anthropic API key.
  • ● Builder Profile — set once, auto-fills across all documents. Business name, ABN, licence, phone, email, address, and authorised representative.
  • ● Document History — all generated documents saved automatically with full load/duplicate/delete functionality.
  • ● Draft Autosave — forms autosave every 1.2 seconds. Drafts survive browser refreshes and can be restored on next visit.
  • ● Beta feedback widget — leave a bug report, idea, feature request, or rating from the banner on every page.
  • ● Trust documents — Privacy Policy, Terms of Use, AI Usage Notice, Support, and Release Notes now published.
Known issues in 0.9 Beta
Known issues
  • PDF print quality may vary between browsers. Chrome and Edge produce the best results. On Safari, use File → Print for best layout.
  • Quote Builder line items may occasionally wrap awkwardly across PDF page breaks on very long quotes. Workaround: use Edit document mode to adjust spacing before printing.
  • Defect severity badge colours may not print on some printers with colour printing disabled. Check print preview before sending.
  • Document history is limited to this browser and device. There is no cloud sync in Version 0.9.
